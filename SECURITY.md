# Security Policy

## Reporting

Please open a private security advisory on GitHub for vulnerabilities.

## Scope

Security-sensitive areas include Accessibility permission handling, global event taps,
emergency stop behavior, release automation, and dependency updates.

## Safety Rules

- Always keep a timer-based automatic stop.
- Keep an emergency keyboard chord outside the blocked path.
- Do not market the app as a security lock or kiosk system.
- Do not collect input event contents.
